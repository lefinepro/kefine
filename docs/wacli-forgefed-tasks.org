#+TITLE: wacli ForgeFed Tasks Integration
#+DATE: 2026-02-21
#+STARTUP: overview

* Overview
Integration of **wacli** with **Crater** (ForgeFed proxy service) for managing tasks (Ticket/Issue) via ActivityPub.

* Quick Start

#+BEGIN_SRC bash
# Initialize tool
wacli ain server-api.example.org

# Authentication (HTTP Signatures for ForgeFed)
wacli auth server-api.example.org --key ~/.wacli/forgefed-key.pem

# Create ticket interactively
wacli server-api.example.org create

# Create ticket from Org file
wacli server-api.example.org create file.org

# List all tickets (table output)
wacli server-api.example.org tasks

# Update ticket
wacli server-api.example.org update 123 --status closed
#+END_SRC

* wacli Commands

** create - Create Ticket

*** Interactively
#+BEGIN_SRC bash
wacli server-api.example.org create
#+END_SRC

Interactive mode will prompt:
1. Ticket kind (bug/feature/support/task/review)
2. Title
3. Description (Markdown, multiline)
4. Priority (low/medium/high/critical)
5. Labels (comma-separated)
6. Milestone (optional)
7. Due date (YYYY-MM-DD)

*** From File
#+BEGIN_SRC bash
wacli server-api.example.org create file.org
#+END_SRC

Org file format:
#+BEGIN_SRC org
* Bug: Login page crashes on mobile
:PROPERTIES:
:kind: bug
:priority: high
:labels: bug,mobile,critical
:milestone: v1.0
:due_date: 2026-03-01
:END:

** Description
Steps to reproduce:
1. Open app on mobile
2. Navigate to login page
3. App crashes

** Expected behavior
App should display login form

** Actual behavior
App crashes with segmentation fault
#+END_SRC

* ForgeFed Integration

** ActivityPub Mapping

| wacli Command | ForgeFed Activity | Description |
|---------------|-------------------|-------------|
| ~create~ | ~Offer~ (Ticket) | Create ticket |
| ~update~ | ~Edit~ | Update metadata |
| ~close~ | ~Resolve~ | Close ticket |
| ~reopen~ | ~Undo~ (Resolve) | Reopen ticket |
| ~assign~ | ~Assign~ | Assign ticket |
| ~delete~ | ~Delete~ | Delete ticket |
| ~comment~ | ~Create~ (Note) | Add comment |

** HTTP Signatures

ForgeFed uses HTTP Signatures for authentication:

#+BEGIN_SRC bash
# Generate key
openssl genrsa -out ~/.wacli/forgefed-key.pem 2048

# Authenticate
wacli auth server-api.example.org --key ~/.wacli/forgefed-key.pem

# Verify
wacli server-api.example.org tasks
#+END_SRC

** Federation Flow

#+BEGIN_SRC text
┌──────────┐    Offer (Ticket)     ┌───────────────┐
│  wacli   │ ────────────────────→ │  Crater       │
│  (CLI)   │                       │  (Proxy)      │
│          │ ← ─ ─ Accept/Reject ─ │               │
└──────────┘    result = issue URI └───────────────┘
       │                                  │
       │         ActivityPub/Fediverse    │
       └────────────────────────────────→ │
                                          │
                                  ┌───────┴───────┐
                                  │   Federated   │
                                  │   Servers     │
                                  │   (ForgeFed)  │
                                  └───────────────┘
#+END_SRC

wacli server-api.example.org assign 123 --assignee_id=user-me
#+END_SRC

** 4. Close Ticket

#+BEGIN_SRC bash
wacli server-api.example.org close 123
#+END_SRC

** 5. Add Comment

#+BEGIN_SRC bash
wacli server-api.example.org comment 123 --file=fix-explanation.md
#+END_SRC

** 6. Export to Org

#+BEGIN_SRC bash
wacli server-api.example.org task 123 --render=org > task-123.org
#+END_SRC
